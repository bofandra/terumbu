# Next Page: Campaign Detail Page

The next page should be the **Campaign Detail Page**, because it is the main conversion page between the homepage and donation checkout.

Its primary job is to answer:

1. What problem are we solving?
2. Who is responsible for the project?
3. Can I trust this campaign?
4. What impact will my contribution create?
5. What happens after I donate?
6. How can I contribute immediately?

---

# 1. Page Objective

## Primary conversion

Encourage users to donate.

## Secondary conversions

* Sponsor a specific coral
* Share the campaign
* Save the campaign
* Follow project updates
* Join a related expedition
* Explore the implementing NGO

## UX principle

The page should move users through:

**Emotion → Evidence → Trust → Contribution → Continued engagement**

# 11. Campaign Story Section

The story should be concise, visual, and structured.

Avoid one long article.

## Section A: The problem

Headline:

```text
Raja Ampat’s reefs need urgent restoration
```

Content:

* Current reef damage
* Main causes
* Effect on marine biodiversity
* Effect on coastal communities

Use:

* 2–3 short paragraphs
* One supporting statistic
* One strong image

## Section B: The solution

Headline:

```text
A restoration program led by local conservation teams
```

Explain:

* Coral nursery development
* Fragment transplantation
* Reef monitoring
* Community participation
* Education and livelihood support

## Section C: Why this location

Include:

* Ecological importance
* Local community relevance
* Existing restoration capacity
* Long-term monitoring access

* Campaign location
* Restoration zones
* Coral nursery
* Monitoring points

# 19. Verification and Governance

Add a dedicated trust panel.

## Verification checklist

```text
✓ Legal organization verified
✓ Bank account verified
✓ Project proposal reviewed
✓ Budget reviewed
✓ Field location validated
✓ Reporting schedule approved
```

## Verification levels

Possible model:

### Verified

Document verification completed.

### Field Verified

Location or project implementation independently checked.

### Impact Verified

Reported outputs validated after implementation.

Use clear definitions and avoid creating the impression that every result is independently audited unless it is.

# 20. Financial Transparency

## Budget breakdown

Use a donut chart or horizontal bars.

Example:

```text
Coral nursery and materials       35%
Field implementation              25%
Monitoring and documentation      15%
Community training                10%
Logistics                          8%
Platform and payment costs         5%
Contingency                         2%
```

Show exact amounts as well as percentages.

## Expandable detailed budget

Users should be able to open:

```text
View Detailed Budget
```

Detailed table:

| Budget item        | Quantity |   Unit cost |         Total |
| ------------------ | -------: | ----------: | ------------: |
| Coral frames       |      100 | Rp1,500,000 | Rp150,000,000 |
| Monitoring dives   |       12 | Rp4,000,000 |  Rp48,000,000 |
| Community training |        4 | Rp8,000,000 |  Rp32,000,000 |

## Platform fee disclosure

Clearly state:

```text
Platform fee: 5%
Payment processing fee: depends on payment method
```

Transparency should be visible before checkout, not hidden in terms and conditions.

# 31. Empty, Loading, and Error States

## Loading

Use skeleton screens for:

* Campaign statistics
* Images
* Updates
* Map
* Donation widget

## Campaign fully funded

Replace the primary message with:

```text
Funding Goal Reached
```

Actions:

* Join waiting list
* Follow implementation
* Support a related campaign

## Campaign ended

Show:

* Final amount
* Final number of donors
* Project status
* Final or latest report
* No active donation CTA

## Campaign paused

Display a clear notice:

```text
Contributions are temporarily paused while this campaign is under review.
```

## Failed data load

Keep core campaign information visible and provide retry actions.

# 33. Accessibility

Minimum requirements:

* WCAG AA color contrast
* Keyboard-accessible tabs and accordions
* Visible focus states
* Alt text for campaign evidence
* Captions for videos
* No information conveyed by color alone
* Screen-reader labels for progress bars
* Donation amounts readable without visual icons
* Map content also available as a text list
* Motion-reduction support

Example progress label:

```text
Campaign is 72 percent funded, with Rp250 million raised toward a Rp350 million goal.
```

* Real conservation photography

### Primary

Coral orange:

```text

```

### Secondary

Ocean blue:

