## Overall assessment

The homepage now matches the reference content architecture closely: header, cinematic hero, three CTAs, impact counters, three campaign cards, featured destination rail, interactive impact-map preview, how-it-works journey, community stories, volunteer path, passport preview, partner strip, and footer are represented.

The implementation remains a production web page rather than an exact 16:9 pitch-deck recreation. It now captures most of the screenshot's first-viewport intent through a full-height hero, social proof, icon counters, destination rail, richer map preview, and partner-logo treatment. Remaining deviations are mostly asset or product-dependency choices: no real hero video asset yet, no Mapbox dependency, limited seeded expedition count, and no true before/after photo dataset for map pins.

## Changes implemented

- Header now shows visible `Login` text, `Sign Up / Join`, `Community`, and `About Us`.
- Hero now uses `min-h-screen`, has a cinematic fallback ocean image, keeps all three CTAs, fixes `Join an Expedition`, and adds social proof plus `Watch impact`.
- Impact counters now include icons.
- Featured expeditions now render as a horizontal destination rail with image cards, ratings, availability, price, and a `More trips` card.
- Impact map pins are now clickable and keyboard-focusable, with a selected-site detail panel and Donate link to the related campaign when available.
- Map-site data now includes `campaignSlug`.
- Community stories, stay-updated CTA, and volunteer/community paths were added.
- Passport preview now has a designed fallback when no public passport exists.
- Partner section now uses `getHomepagePartners()` and renders a logo/monogram strip with organization and corporate partner support.
- Empty states were added for map sites and expedition rail.

## Remaining gaps

1. Add a real cinematic video asset for the hero, with the current image as poster/fallback.
2. Add actual organization logo assets to seed/admin data so the partner strip can render real NGO and community logos.
3. Add at least one more seeded expedition destination if the homepage should always show three destination cards without relying on the `More trips` card.
4. Add real before/after media to impact-site or evidence data if map pin popups need photo comparison.
5. Decide whether to adopt Mapbox. OpenStreetMap is intentionally retained for now to avoid a new dependency and token requirement.
6. Add animated counter behavior if the live-counter motion is important for final visual fidelity.
7. Consider expanding the how-it-works sequence to the screenshot's five explicit steps.
