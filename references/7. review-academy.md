## Overall assessment

The Academy implementation is now much closer to the reference and screenshot. The public Academy home has moved from a thin course grid to a learning gateway with an image-backed hero, prominent search, popular topics, trust row, personalized learner summary, continue-learning area, learning tracks, expedition-preparation panel, live-session card, certificate preview, filterable catalog, learning-to-action journey, instructor cards, footer trust strip, and route-level loading/error states.

The course detail page is also substantially stronger. It now reads as a course landing/player surface with hero imagery, course facts, enrollment/progress panel, lesson outline, completion requirements, resources, certificate language, related action links, and a final-check flow that unlocks after lessons are complete.

The remaining gaps are deeper product models and workflows rather than the first-screen experience: persisted learning tracks, real instructor/session/resource/review tables, full quiz questions and retake policy, saved courses, transcript exports, live-session registration, certificate download/share verification, field-eligibility expiry, corporate assignments, and richer analytics.

## Changes implemented in this pass

- Added `getAcademyHomeData()` to shape Academy homepage data from existing courses, lessons, enrollments, progress, certificates, organizations, and expedition bookings.
- Derived first-version course metadata, learning tracks, learner summary, continue-learning, preparation modules, live session, certificate preview, instructors, and trust stats without adding migrations.
- Rebuilt `/academy` around the screenshot hierarchy: hero search, popular topics, trust indicators, learner progress, continue learning, tracks, expedition prep, catalog filters, course cards, live learning, certificate preview, journey, instructors, and final CTA.
- Rebuilt `/academy/courses/[slug]` with hero imagery, course facts, progress, lesson status, completion requirements, resources, certificate framing, and improved assessment unlock behavior.
- Added Academy-specific loading and error states for home and course detail routes.
- Preserved existing enrollment, lesson completion, assessment, certificate issuance, and Impact Passport integration behavior.

## Existing strengths

- Public `/academy` route exists and lists database-backed courses.
- Public course detail route exists at `/academy/courses/[slug]`.
- Course enrollment is implemented through `enrollCourseAction()`.
- Lesson completion is implemented through `completeLessonAction()`.
- Final assessment submission is implemented through `submitAssessmentAction()`.
- Passing an assessment marks enrollment complete, creates a certificate, and adds an Impact Passport item when the user has a passport.
- Dashboard Academy page shows enrollments and certificates.
- Dashboard overview has a continue-learning card with progress, next lesson context, minutes remaining, and learning stats.
- Database schema covers the MVP learning core: courses, lessons, enrollments, lesson progress, assessments, attempts, and certificates.

## Screenshot comparison

The screenshot target shows a rich logged-in Academy homepage with dark hero, search, trust row, learner summary, continue-learning card, learning tracks, expedition prep, featured courses, live class, certificate preview, journey, instructors, and trust strip.

The implementation now matches that page structure at a first-version fidelity level. The visual details are not pixel-perfect, and several modules are derived/static because the deeper Academy content models do not yet exist, but the information architecture and user journey now align with the reference.

## Remaining highest-priority gaps

1. Add persisted Academy product models: learning tracks, track-course mapping, instructors, live sessions, resources, saved courses, reviews, and transcript exports.
2. Replace the final-check shortcut with a real quiz/assessment model: questions, answers, attempts, retake limits, feedback, accessibility support, and audit trail.
3. Link courses to campaigns and expeditions so donor education and required trip preparation are data-backed.
4. Add certificate download/share/public verification and full learning transcript exports.
5. Add corporate learning assignments, internal challenges, employee progress, and sponsor access rules.
6. Add search suggestions, sorting, more filters, grouped results, saved courses, and learner stories.
7. Add Academy analytics events for search, enrollment, lesson completion, assessment, certificate issuance, and learning-to-action conversion.
