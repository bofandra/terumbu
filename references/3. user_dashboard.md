# Next Page: User Dashboard / Impact Hub

The next page should be the **User Dashboard**, because it closes the loop after a user donates, joins an expedition, completes a course, or sponsors a coral.

The dashboard should make users feel:

> “My contribution is real, visible, growing, and worth continuing.”

Its experience should combine the best parts of:

* A banking dashboard for clear activity and transactions
* Strava for progress and achievements
* LinkedIn for the public Impact Passport
* A travel app for upcoming expeditions
* A conservation monitoring platform for project evidence

---

# 1. Main Objectives

## Primary objective

Help users understand their total conservation impact.

## Secondary objectives

* Encourage repeat donations
* Show sponsored coral progress
* Manage expedition bookings
* Continue academy courses
* Download certificates and receipts
* Follow campaign updates
* Build and share an Impact Passport
* Increase user level and achievements

## UX flow

```text
Review impact
→ Check new updates
→ Continue an activity
→ Discover the next action
→ Share progress
```

# 3. Desktop Layout

Use a fixed left sidebar and a responsive main content area.

## Recommended proportions

* Sidebar: 240–270 px
* Main content: remaining width
* Maximum content width: 1,440 px
* Main grid: 12 columns
* Card radius: 16–20 px
* Card gap: 20–24 px

## General layout

```text
┌──────────────┬──────────────────────────────────────────┐
│ Sidebar      │ Header / user controls                   │
│              ├──────────────────────────────────────────┤
│ Home         │ Welcome + Ocean Hero level               │
│ My Impact    ├──────────────────────────────────────────┤
│ Corals       │ Impact summary cards                     │
│ Donations    ├───────────────────┬──────────────────────┤
│ Expeditions  │ Impact map        │ Latest update        │
│ Academy      ├───────────────────┴──────────────────────┤

│              ├───────────────────┴──────────────────────┤
│ Support      │ Activity timeline and achievements       │
└──────────────┴──────────────────────────────────────────┘
```

# 10. Personal Impact Map

This should be one of the dashboard’s signature components.

## Map displays

* Donated campaign locations
* Sponsored coral locations
* Completed expeditions
* Upcoming expeditions
* Volunteer activities
* Academy field activities

## Pin categories

* Blue: coral restoration
* Green: mangrove
* Orange: ocean cleanup
* Purple: community program
* Turquoise: expedition

## User interaction

Clicking a pin opens:

```text
Raja Ampat Coral Restoration

Rp1.2M contributed
20 coral fragments sponsored
1 expedition completed
Latest update: 18 June 2026
```

CTA:

```text
View My Impact Here
```

## Privacy and ecology

Exact ecological coordinates should be hidden where public disclosure could harm the site.

Use approximate zones with a note:

```text
Location shown approximately to protect the restoration area.
```

## Alternative accessible view

Provide a “List View” option for users who cannot use the map.

# 22. User Profile Completeness

Show only when relevant.

Example:

```text

```

* Emergency contact
* Dietary preference
* Diving certification
* Identification document
* T-shirt size

# 28. Error States

## Partial data failure

Do not block the whole dashboard.

Example:

```text
We could not load your impact map.
Your contribution data is still available below.
```

## Payment problem

Use a clear alert:

```text
Your monthly contribution payment failed.

Update your payment method to continue supporting the campaign.
```

CTA:

```text
Update Payment Method
```

## Data under review

For impact metrics:

```text
Some project data is being verified and may change after review.
```

# 29. Accessibility

Minimum requirements:

* WCAG AA contrast
* Full keyboard navigation
* Visible focus indicators
* Screen-reader labels for charts
* Text alternatives for the map
* Statuses not identified by color alone
* Reduced-motion option
* Descriptive image alt text
* Progress bars with numerical labels
* Accessible carousel controls
* Minimum mobile touch target of 44 × 44 px

Example chart description:

```text
Your monthly contributions increased from Rp100,000 in January to Rp500,000 in June.
```

---

# 30. Privacy Controls

The dashboard contains personal and potentially sensitive activity information.

## User controls

* Hide total donation value
* Hide sponsored coral count
* Hide expedition history
* Hide achievements
* Make passport private
* Use anonymous public name
* Control profile search visibility

## Public versus private content

Private dashboard:

* Payment details
* Receipts
* Booking documents
* Exact activity records

Public Impact Passport:

* Approved impact summary
* Optional activities
* Selected achievements
* Approximate locations only

---

# 31. Visual Design Direction

Continue the design language from the homepage and campaign detail page.

## Base colors

* Deep navy for navigation
* Ocean blue for links and data
* Coral orange for primary actions
* Mangrove green for positive status
* Soft aqua backgrounds for impact cards
* White for primary surfaces

## Card style

* White background
* Thin neutral border
* Soft shadow
* Rounded corners
* Generous spacing

## Imagery

Use actual conservation evidence wherever possible:

* Field photos
* Monitoring photos
* Site images
* Expedition documentation

Avoid filling the dashboard with generic stock photos.

---

# 32. Data Visualization Rules

## Use charts for

* Trends over time
* Distribution of contributions
* Impact category breakdown
* Learning progress

## Use numbers for

* Total donations
* Corals sponsored
* Courses completed
* Expeditions joined

## Use maps for

* Geographic activity
* Restoration locations
* Expedition history

## Avoid

* Decorative charts without clear decisions
* Overly complex dashboards
* Estimated environmental claims without methodology
* Too many competing metrics

# 36. Primary Success Metrics

Track:

* Dashboard return rate
* Latest-update open rate
* Repeat donation conversion
* Course continuation rate
* Expedition-preparation completion rate
* Impact Passport share rate
* Sponsored-coral detail views
* Monthly report downloads
* Recommendation click-through rate

The dashboard succeeds when users return to **see what changed**, not only when they return to spend money.
